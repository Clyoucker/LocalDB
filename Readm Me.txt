В общем, смысла в этой программе очень мало, но я сделал от нефиг делать.

Команды:

Создание категории
create.category(category="category_name",attributes="link:=str developer:=str")
В атрибутах пишите название переменной, а потом при помощи := присваиваете ей тип переменной.

LocalDB поддерживает следующие типы переменных:
    String (строка) - str
    Integer (целочисленный) - int
    Float (дробное) - float
    List (список) - list
    Dict (словарь - ключ:значение) - dict
    Time (тут имеется ввиду секунды,минуты и часы, а не дата с временем) - time
    Date (тут уже дата с временем) - date
    Href (ссылочный тип. Автоматический подставит https:// если такого нет в значении) - href "WARNING! Work In Progress"
    Money (денежный формат) - money "WARNING! Work In Progress"

Для добавления элемента в категорию, исп. команда create.obj.
create.obj(attributes:="developer:=clyoucker sites:=libk1,link2",category:="category_name")
Категорию при добавлении объекта можно не писать, если в неё уже 1 раз что-то вводилось. Она автоматически подставится, если же такого не надо, просто пишите
категорию в которую хотите добавить.

В LocalDB есть множественное удаление для удобства. Если просто написать одно значение, то это будет одно удаление, ну а иначе несколько.
del.obj(obj_id=(1-5,8),category=Arts)
ТИРЕ для диапазона удаления. ЗАПЯТАЯ для других id.

Также в LocalDB есть константы, ключи и шаблоны.
Ключи нужны для проверки, что переменная задана.
Константы же для неизменяемых значений. То есть, вы не сможете при помощи команды change.obj изменить в нём константное значение.
Шаблоны - для автоматического присваивания значения, если переменная при создании объекта пустая.

Если при создании категории можно сразу же создать константу/шаблон или ключ, но необходимо исп. следующие выражения.
При создании константы:
    create_category(category="Developers",attributes="@title:=str")
При создании шаблона:
    create_category(category="Developers",attributes="!title[no-title]:=str")
При создании ключа:
    create_category(category="Developers",attributes="$title:=str")

Возможно когда-нибудь я добавлю связи между категориями.

Я это использовал для React приложения, ибо лень было создавать полноценную БД, так как это был тестовый проект.
Вы же можете это использовать для своих каких-то целей.
